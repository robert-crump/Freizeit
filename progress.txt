## 2026-07-05 — Issue #3: POI import end-to-end (extraction file → Room, keep+flag)

- Schema (still Room v1 — no installed devices exist, so no migration needed):
  - Poi evolved: String id ("node/123") PK, category, lat/lon, optional
    name/opening_hours/address fields, missingFromOsm flag.
  - Verdict table added now (storage/UI lands in #6) because the import's keep+flag
    rule depends on it; snapshots name/coords/category, deliberately no FK to poi.
  - ImportInfo single-row table (importedAt, fileGeneratedAt); per-category counts
    are queried live (GROUP BY), not stored.
- PoiJsonParser (Gson): strict validation of the tools/poi_extraction format;
  whole file parses before any DB write.
- PoiRepository.importFrom(uri) in one transaction:
  markAllMissing → upsert file contents (clears flag) → delete missing-without-verdict.
  Malformed/unreadable file throws PoiParseException, database untouched.
- Settings screen: SAF file picker (OpenDocument), import progress/success/error
  states, summary card with per-category counts, last-import timestamp, and
  kept-but-missing count. SettingsViewModel with manual DI factory.
- Tests (17 total, all green): parser unit tests + Robolectric in-memory-Room tests
  covering all five acceptance criteria (all categories import, idempotent re-import,
  unverdicted vanished place dropped, verdicted one kept+flagged and un-flagged on
  return, malformed file leaves DB untouched).
- Gotcha: Room @Upsert breaks under Robolectric's legacy SQLite shadow (Windows) —
  switched DAOs to @Insert(onConflict = REPLACE), which is also the semantics the
  import wants (full-row replace).
- Device still disconnected; on-device SAF-picker flow untested. Build + full test
  suite green.

## 2026-07-05 — Hotfix: Settings crashed on first import (reported by Robert)

- Cause: return@Column early-return in ImportSummaryCard's empty state; when the
  summary flow emitted after the first successful import, the branch switch
  corrupted the composer group stack → IndexOutOfBoundsException at Stack.pop.
- Fix (d87b81a): restructured as if/else; grep confirmed no other return@ inside
  composables. Rule: no early returns inside composable lambdas.

## 2026-07-06 — Issue #5: Home — three suggestion cards with transparent ranking and weather

- domain/ layer (pure Kotlin, fully unit-tested, no Android deps):
  - OpeningHours: conservative OSM opening_hours evaluator implementing the #1
    hybrid decision — only ever returns CLOSED when the value is fully understood
    (24/7, day ranges/lists, split + overnight spans, "off", ";" rule override,
    PH rules skipped); anything else is UNKNOWN = never filter by hours.
  - WeatherSnapshot: current + 12h hourly, WMO code helpers, badWeatherWithin /
    dryHoursAhead / outlook for the strip.
  - SuggestionEngine: hard filters (known-closed, outdoor when rain falls inside
    the outing window, unreachable = 2×bike travel + 30 min stay > budget; bike =
    Haversine ×1.3 detour at 250 m/min) then transparent soft score (distance
    decay 40·e^(-min/25), weather-fit bonuses, +6 confirmed-open, deterministic
    daily novelty jitter 0–8) × who-is-along category weight (kids default
    neutral; adults-only ready for #7). Diversity: if the top 3 share a category
    the third is swapped for the best other-category place. rankAll() exposed for
    tests/reroll; reason line parts ("Open", "4 min by bike", "dry all day",
    "21° ice-cream weather", "good for a rainy day") built alongside the score.
- data/weather: OpenMeteoClient (plain HttpURLConnection + Gson, keyless, null on
  any failure) + WeatherRepository with stale-beats-spinner semantics: disk cache
  in filesDir served instantly, refetch only when >30 min old, failures keep the
  cache. Live API response shape verified against the DTO from the desktop.
- ui/home: HomeViewModel (combine POIs + weather + location + reroll exclusions
  on Dispatchers.Default; reroll excludes all session-shown ids and wraps when
  the pool runs dry; Cologne fallback for weather when no location) and the new
